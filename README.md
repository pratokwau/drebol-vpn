# drebol-vpn

Telegram-бот для управления VPN на базе `3x-ui` с отдельными сценариями для админских и платных подписок.

## Что умеет бот

- показывает главное меню через `/start`
- открывает поддержку через кнопку `🆘 Поддержка` и команду `/help`
- управляет админской VPN-подпиской
- управляет платной подпиской и её заявками на выдачу, оплату и продление
- показывает инструкцию с HAPP и ссылкой подписки
- работает с тикетами поддержки
- показывает админское меню только администратору
- присылает уведомление об обновлении при появлении нового коммита

## Основные разделы

### Главное меню `/start`

В стартовом меню кнопки появляются в зависимости от прав и наличия подписки:

- `🔐 Мой VPN` — для пользователей с админской подпиской
- `💳 Моя подписка` — для пользователей с платной подпиской
- `🆘 Поддержка` — доступно всем
- админские кнопки — только для `ADMIN_ID`

### Админская подписка

Раздел для управления обычными VPN-клиентами:

- список пользователей и их устройств
- настройка лимитов устройств, IP, трафика и `flow`
- настройка инбаундов
- синхронизация с панелью `3x-ui`
- просмотр инструкции по подключению

### Платная подписка

Раздел для коммерческой подписки:

- отдельные настройки триала, срока оплаты, суммы и grace-периода
- заявки на первичный триал
- заявки на проверку оплаты
- продление подписки
- заморозка и разморозка подписки
- просмотр карточки подписки и статистики клиента
- отдельная логика для устройств без Telegram-данных

### Поддержка

Через тикеты можно:

- создавать обращение
- отвечать в истории чата
- прикладывать медиа
- смотреть список тикетов из админки

## Инструкция по установке на Ubuntu

### Быстрая установка

```bash
bash <(curl -Ls https://raw.githubusercontent.com/pratokwau/drebol-vpn/main/install.sh)
```

Во время настройки бот попросит:

- токен Telegram-бота
- `ADMIN_ID`
- параметры подключения к `3x-ui`
- URL панели
- API token панели
- порт подписки

После этого установщик:

- создаст `.env`
- создаст/обновит файлы в `data/`
- установит зависимости
- поднимет сервис `systemd`
- включит автозапуск

## Полезные команды

### Пользовательские

- `/start` — главное меню
- `/sub` — меню платной подписки
- `/help` — поддержка
- `/cancel` — отмена текущего действия

### Админские

- `/admin` — админское меню
- `/adminxui` — старый вход в админское меню, работает как alias
- `/adminsub` — управление админскими VPN-подписками
- `/adminpaysub` — управление платными подписками
- тикеты доступны из админского меню

## Как устроен проект

- `main.py` — точка входа, фоновые проверки и уведомления
- `handlers/` — старт, админка, поддержка, отмена
- `sub/adminsub/` — логика админских подписок и 3x-ui
- `sub/adminpaysub/` — логика платных подписок
- `sub/instructions.py` — генерация инструкции и ссылок подписки
- `sub/api.py` — работа с API панели
- `storage.py` — общие JSON-хранилища
- `data/` — данные пользователей, заявок и настроек

## Данные и хранилища

Проект хранит состояние локально в JSON-файлах:

- `data/authorized.json`
- `data/settings.json`
- `data/xui_settings.json`
- `data/vpn_users.json`
- `data/paid_settings.json`
- `data/paid_subscriptions.json`
- `data/paid_requests.json`
- `data/tickets.json`

## Важные примечания

- бот работает через `long polling`
- для работы нужен корректный Telegram Bot Token
- `ADMIN_ID` должен быть числом
- для подключения к панели нужны корректные `XUI_URL` и `XUI_TOKEN`
- ссылка подписки формируется из URL панели и порта подписки
- инструкция использует приложение HAPP для Android, iOS, macOS и Windows

## Обновления

Бот проверяет наличие новых обновлений и уведомляет администратора в Telegram.
